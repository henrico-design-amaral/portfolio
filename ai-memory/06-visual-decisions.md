# VISUAL DECISIONS — Portfolio_v2

Este arquivo registra decisões visuais específicas do portfólio Portfolio_v2.

## Status

Em evolução ativa.

As decisões abaixo devem ser validadas contra o código atual antes de alterações profundas.

## Direção visual aprovada

O portfólio deve seguir uma estética:

- editorial
- técnica
- madura
- sofisticada
- sóbria
- precisa
- orientada a sistemas
- com aparência autoral
- sem aparência de template genérico

## Decisões consolidadas

### 1. O portfólio não deve parecer landing page genérica

Evitar composição visual comum de SaaS, excesso de cards genéricos, gradientes sem função e blocos de conteúdo previsíveis.

### 2. As seções precisam conversar entre si

Hero, Método, Cases e CTA devem parecer partes de um mesmo sistema.

Se uma seção destoar, a decisão deve ser:

- adaptar a seção ao sistema existente
ou
- evoluir o sistema inteiro de forma consistente

Nunca deixar seções conflitantes convivendo.

### 3. Backgrounds devem ser sutis

Backgrounds podem ter grid, ruído, textura, profundidade e motion.

Mas não devem competir com o conteúdo.

### 4. Grid precisa ser controlado

Grid é recurso de estrutura.

Não deve parecer elemento decorativo dominante.

Se o grid de uma seção estiver mais forte que o restante do site, deve ser suavizado ou substituído por superfície mais sólida.

### 5. Motion deve ser sofisticado

Motion deve ser sutil e quase imperceptível em backgrounds.

Usar motion para dar vida e continuidade, não para chamar atenção.

### 6. Imagens dos cases devem ter função narrativa

As imagens dos cases devem reforçar a natureza do projeto.

Elas não devem ser apenas imagens bonitas.

Devem comunicar:

- Petrobras: operação crítica offshore, coordenação, monitoramento, decisão
- Bayer: dados, interpretação, agricultura, análise, suporte técnico
- Ambev: logística, fluxo, operação, monitoramento
- BMG: produto financeiro, conta PJ, serviços modulares, confiança

## Estilo visual a evitar

Evitar:

- excesso de neon
- cyberpunk
- sci-fi genérico
- dashboard futurista exagerado
- glassmorphism sem função
- gradientes saturados
- grid muito evidente
- sombras pesadas
- interfaces decorativas falsas
- visual de template premium genérico

## Estilo visual preferido

Preferir:

- superfície escura ou neutra bem controlada, se compatível com o projeto atual
- contraste sofisticado
- detalhes técnicos sutis
- composição editorial
- tipografia forte
- cards com intenção
- imagens cinematográficas, mas plausíveis
- motion lento
- atmosfera institucional e madura

## Pontos de atenção

### Cases

A seção de cases precisa estar alinhada ao restante do site.

Problemas possíveis:

- fundo destoando da Hero
- grid forte demais
- cards com densidade incoerente
- imagens competindo com texto
- textos excessivamente técnicos
- falta de hierarquia

### Método

A seção Método deve explicar o raciocínio de Henrico de forma clara, sem linguagem excessivamente técnica ou artificial.

### Hero

A Hero deve ser a referência de tom, atmosfera e maturidade visual do site.

## Regra final

Toda decisão visual deve reforçar o posicionamento profissional de Henrico.

Se a decisão apenas deixa o layout mais bonito, mas não reforça o posicionamento, ela é fraca.
