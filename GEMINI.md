# GEMINI.md — Antigravity Rules for Portfolio

Ao trabalhar neste projeto, leia e siga obrigatoriamente:

- PROJECT_CONTROL.md
- AGENTS.md

## Procedimento obrigatório no início de cada sessão

Antes de alterar qualquer arquivo:

1. verificar git status
2. verificar branch atual
3. ler PROJECT_CONTROL.md
4. ler AGENTS.md
5. identificar arquivos relevantes
6. apresentar diagnóstico curto
7. apresentar plano de ação
8. aguardar aprovação humana

## Regra central

Não execute mudanças amplas sem plano.

Não altere arquivos fora do escopo solicitado.

Não trabalhe direto na main para alterações estruturais, visuais grandes ou experimentais.

## Fluxo de resposta esperado

Antes da execução, responder com:

1. estado atual
2. arquivos que serão afetados
3. riscos
4. plano curto
5. critério de aceite

Depois da execução, responder com:

1. arquivos modificados
2. resumo do que mudou
3. como testar
4. riscos restantes
5. sugestão de commit

## Padrão de qualidade

Este portfolio deve parecer editorial, técnico, sóbrio e sênior.

Priorizar:
- clareza
- estrutura
- consistência visual
- narrativa progressiva
- performance
- acessibilidade
- manutenção simples

Evitar:
- excesso visual
- alterações decorativas
- soluções genéricas
- copy inflada
- mudanças difíceis de reverter
