# Homepage Refinement Skill

## Purpose

Refine the portfolio home page without breaking positioning, accessibility, performance or visual coherence.

## Scope

Use this skill when editing:

- `index.html`;
- home page section copy;
- home page layout;
- hero;
- method/process section;
- cases section;
- impact section;
- about section;
- contact/footer.

## Required agents

Recommended:

- `portfolio-orchestrator`;
- `portfolio-home-strategist`;
- `frontend-implementer`;
- `git-pr-reviewer`.

## Required companion skills

Recommended:

- `homepage-quality-audit`;
- `anti-ai-slop-editorial-review`;
- `design-taste-review`;
- `html-page-structure-guard`;
- `visual-regression-check`.

## Rules

- Preserve PT-BR and EN-US.
- Preserve semantic HTML.
- Keep CSS in `assets/css/`.
- Keep JS in `assets/js/`.
- Do not add unnecessary libraries.
- Do not change performance setup unless explicitly asked.
- Do not change unrelated case pages.
- Do not commit without explicit instruction.

## Quality focus

Evaluate:

- clarity above the fold;
- positioning within 30 seconds;
- section rhythm;
- case card strength;
- method clarity;
- impact credibility;
- CTA clarity;
- mobile readability;
- visual refinement;
- language quality.

## Output

Before implementation, report:

- diagnosis;
- proposed changes;
- files likely to be touched;
- risks.

After implementation, report:

- files changed;
- semantic structure preserved;
- CSS/JS organization;
- bilingual implementation;
- visual and accessibility risks;
- recommended validation.
