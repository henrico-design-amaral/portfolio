# Case Page Foundation Skill

## Purpose

Create the foundation for an individual portfolio case page.

## Procedure

1. Confirm branch.
2. Inspect `index.html`, `assets/css/site.css` and existing case cards.
3. Create or update only the case page.
4. Use existing CSS and visual language.
5. Do not add libraries.
6. Do not alter global JS.
7. Do not invent metrics or fake screenshots.
8. Validate local rendering before commit.

## Output

Report files changed, structure created, risks and next step.
